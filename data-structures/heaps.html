Heaps


A heap is essentially a type of complete binary Tree. In a min-heap, each node is smaller than its children. Likewise, in a max-heap, each node is larger than its children. This means the root node is always the max or min item. Heaps are useful for when you need a way to quickly retrieve or delete the max or min item in a collection. Heaps are a common way of implementing a priority queue, and sometimes priority queues are called heaps.


Min heap (left) and max heap (right)

Implementation


A heap is implemented as a binary tree, with functions unique to a heap.
A Heap class must support two key operations:
  1. insert(item)
    This function inserts an item into the heap. In order to maintain completeness, we start at the bottom level and find the first available spot from left to right. Then, while the new item's parent is greater/less than (this depends on the type of heap) the new item, we swap the two. We continue doing this, gradually moving the new item up the tree until we find the right spot for it.
  2. retrieve_min() (retrieve_max() for a max-heap)
    This function simply grabs the max or min item in the heap, which is always the root. To remove it, we first remove the root and replace it with the bottom-most item. This would be the right-most node on the bottom level. Then we swap the root with its smallest (or largest, depending on the type of heap) child, gradually moving it down the tree until the heap property is fulfilled again.
