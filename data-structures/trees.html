Trees


A tree is actually a type of Graph, with a few distinct properties. In computer science, every tree contains a root node. This root node may contain 0 or more children, and each child may contain 0 or more children of its own. A tree also does not contain any cycles. This means there is no path from any node that leads back to itself. A leaf is a node that does not have any children.
A binary tree is a tree where each node has no more than two children. This is probably the type of tree most computer science students are familiar with. However, a tree can have any number of children per node — 10, 100, 1000 — it doesn't matter. As long as it meets the criteria described above, it is still a tree. A subset of binary trees are binary search trees. Binary search trees are binary trees such that each left descendant is less than its ancestor and each right descendant is greater than its ancestor.


A Binary Search Tree


Implementation


A tree is implemented with a Node class, which contains a list or array of its children. In the case of binary tree, typically you will just have a pointer to its left and right nodes.
You can also create a Tree wrapper class which only contains the root Node.

Examples


Below is an implementation of a Binary tree.

                
// C++
class Node {
  public:
    Node* left;
    Node* right;
    int data; 
}
class Tree {
  public:
    Node* root;
};
                
              
                
# Python
class Node:
  def __init__(self, data):
    self.data = data
    self.left = None
    self.right = None

class Tree:
  def __init__(self, root):
    self.root = root

Binary Tree traversal


Often times you will be asked to traverse a binary tree. This means to visit each node in a particular order. Often times by "visit" we mean print the data at a node. There are three types of binary tree traversals: In-Order, Post-Order, and Pre-Order. We will go through each of them.
  1. In-Order
    Visit the left subtree, then the current node, then the right subtree. 
    2. 
// C++
void inOrder(Node* root) {
  if (root != null) {
    inOrder(root->left);
    visit(root);
    inOrder(root->right);
  }
}
                
    
# Python
def inOrder(root):
  if root:
    inOrder(root.left)
    visit(root)
    inOrder(root.right)
  2. Pre-Order
    Visit the current node before visiting the subtrees. For binary trees it would current node, then left subtree, then right subtree. 
    3. 
// C++
void preOrder(Node* root) {
  if (root != null) {
    visit(root);
    preOrder(root->left);
    preOrder(root->right);
  }
}
                
    
# Python
def preOrder(root):
  if root:
    visit(root)
    preOrder(root.left)
    preOrder(root.right)
  3. Post-Order
    Visit the current node after visiting the subtrees. For binary trees it would be left subtree, then right subtree, then current node. 
    4. 
// C++
void postOrder(Node* root) {
  if (root != null) {
    postOrder(root->left);
    postOrder(root->right);
    visit(root);
  }
}
                
    
# Python
def postOrder(root):
  if root:
    postOrder(root.left)
    postOrder(root.right)
    visit(root)
