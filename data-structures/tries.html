Tries


A trie is a type of Tree, which is a type of Graph. You may sometimes see them be referred to as prefix trees. Tries were invented as a solution to a very specific problem, which was storing and retrieving strings.
A trie is a variant of an n-ary tree in which characters are stored at each node. Each path down the tree may represent a word.
McDowell, "Cracking the Coding Interview"
Typically there will be a way to indicate the end of a word. These can be implemented as a special child such as TerminatingTrieNode or by storing a boolean flag in the parent, such as terminates (McDowell, "Cracking the Coding Interview").
Tries are a very common data structure to be used whenever some type of word validation is needed. A common use case would be implementing an autocomplete function.


A Trie with terminating nodes
"Cracking the Coding Interview", p. 105


Implementation


A trie is implemented in a very similar way to any other tree, but with a slight twist. Each Node contains a Hash table of its children that maps from a key (character) to its corresponding Node. The simplest way to indicate when a word terminates would be to use a boolean terminates flag set to true for any Node that completes a word.

Examples



// C++
class Node {
  char key;
  HashMap<char, Node> children;
  bool terminates;
}
              

# Python
class Node:
  def __init__(self, key, flag):
    self.key = key
    self.terminates = flag
    self.children = {}
