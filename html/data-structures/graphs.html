Graphs


A graph is a collection of nodes connected by edges. A graph can be either directed or undirected. In a directed graph, you can only traverse an edge in a certain direction. In an undirected graph, you can traverse an edge in either direction. A graph with edges between every pair of vertices is called connected. A graph without cycles is known as acyclic.

Implementation


There are two ways to implement a graph. With an adjacency matrix or an adjacency list.
  1. Adjacency Matrix
    The graph is stored as an NxN matrix of booleans, where N is the number of nodes. If there is an edge from i to j, then mat[i][j] is true. Everywhere else is false. This implementation is generally not preferred as you have to iterate through all the nodes to find the neighbors of a node.
  2. Adjacency List
    We use a Node class, which contains a list of all its children. The graph is then represented with a Graph class, which contains a list of all its nodes. This is necessary because it is not always possible to reach every node from a root node, like with a tree.

Examples


Adjacency List 


// C++
class Node {
  char label;
  Node neighbors[];
}
class Graph {
  Node nodes[];
}
              

# Python
class Node:
  def __init__(self, label):
    self.label = label
    self.neighbors = []
class Graph:
  def __init__(self):
    self.nodes = []

Adjacency Matrix 


// C++
class Graph {
  int numNodes;
  int matrix[numNodes][numNodes];
}
              

# Python
class Graph:
  def __init__(self, numNodes):
    self.matrix = [[False for i in range(numNodes)] for i in range(numNodes)]
