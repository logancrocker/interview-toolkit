Hash tables


Hash tables are way of mapping keys to values and are very popular because they provide extremely fast lookup. If you are familiar with Python, a Python dictionary is implemented with a hash table. Hash tables are a good data structure to know because they can often be used for optimization. For example, if you need to keep track of previous strings in an algorithm, you can store them in a hash table to be looked up quickly later. Hash tables are commonly used for storing strings.

Implementation


A hash table is implemented with a linear data structure such as an array, which is indexed via a hash() function. This function takes a key — in our example, a string — and maps it to a value. For example, lets say our hash table has a size of 10 — i.e., its underlying array has a size of 10. Our hash function could then return the sum of all the characters in the key, modulo 10. This would create a value in the range of the array indices that can be used to insert and retrieve it. When we want to retrieve that key (or check if it exists), we just hash it again and then access the array. This is why hash tables have extremely fast lookup and retrieval speeds.


Examples


Key-value maps (dictionaries) are native to Python so I only implemented a simple Hash table in C++. 


// C++
#include <iostream>
#include <string>
  
using namespace std;
  
class Hash {
  private:
    string* keys = new string[100];
  public:
    int hash(string key) {
      int sum = 0;
      for (char c : key) {
        sum += c;
      }
      return sum % 100;
    }
    void insert(string key) {
      int index = this->hash(key);
      this->keys[index] = key;
      return;
    }
    string retrieve(string key) {
      int index = this->hash(key);
      return this->keys[index];
    }
};
